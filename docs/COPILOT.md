# RuleSmith for GitHub Copilot

**Custom Instructions Integration**

---

## Overview

RuleSmith integrates with GitHub Copilot through custom instructions. It analyzes your conversations and generates rule suggestions that Copilot uses to provide more personalized assistance.

---

## Installation

### Method 1: Global Installation

#### macOS / Linux

```bash
# Clone the repository
git clone https://github.com/hariprs/rulesmith.git
cd rulesmith

# Copy skill to global skills directory
mkdir -p ~/.claude/skills/
cp -r .claude/skills/rulesmith/ ~/.claude/skills/
```

#### Windows (PowerShell)

```powershell
# Clone the repository
git clone https://github.com/hariprs/rulesmith.git
cd rulesmith

# Copy skill to global skills directory
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.claude\skills"
Copy-Item -Recurse -Force .claude\skills\rulesmith "$env:USERPROFILE\.claude\skills\"
```

### Method 2: VS Code Extension Integration

#### macOS / Linux

```bash
# For VS Code with Copilot extension
code --install-extension github.copilot

# Copy skill to VS Code workspace
mkdir -p .vscode/.claude/skills/
cp -r .claude/skills/rulesmith/ .vscode/.claude/skills/
```

#### Windows (PowerShell)

```powershell

New-Item -ItemType Directory -Force -Path .vscode\.claude\skills

## GitHub Copilot Configuration

### Setting Up Custom Instructions

1. **Via GitHub Settings:**
   - Go to https://github.com/settings/copilot
   - Navigate to "Custom Instructions"
   - Add the generated rules from RuleSmith

2. **Via VS Code:**
   - Open VS Code Settings
   - Search for "GitHub Copilot"
   - Find "Custom Instructions" section
   - Paste your generated rules

3. **Via Configuration File:**
   ```bash
   # Create custom instructions file
   mkdir -p ~/.config/github-copilot/
   cat > ~/.config/github-copilot/instructions.md << 'EOF'
   # GitHub Copilot Custom Instructions
   # Generated by RuleSmith

   ## Communication Style
   - Keep responses concise and actionable
   - Provide working code examples
   - Explain trade-offs briefly

   ## Code Preferences
   - TypeScript for type safety
   - Modern JavaScript features
   - Clean code principles

   EOF
   ```

## Usage

### Basic Usage

When using GitHub Copilot in VS Code or the web:

```
/rulesmith
```

Or in a Copilot Chat conversation:

```
Please analyze our conversation and suggest improvements to my Copilot instructions.
```

### Applying Generated Rules

After running `/rulesmith`:

1. Copy the suggested rules
2. Paste them into your Copilot Custom Instructions
3. Save and restart your editor

### Example Output

```
📊 Conversation Analysis for GitHub Copilot

**Corrections Found:** 4

**Patterns Identified:**

1. "Prefer functional programming" (count: 2)
   → Copilot Instruction: "When writing JavaScript/TypeScript, prefer functional
   programming patterns: map, filter, reduce over for loops"

2. "More context in comments" (count: 2)
   → Copilot Instruction: "Add explanatory comments for complex logic, explaining
   the 'why' not just the 'what'"

**Instructions to add to Copilot:**
```markdown
## Code Style Preferences
- Prefer functional programming patterns: map, filter, reduce over loops
- Add explanatory comments for complex logic (focus on 'why')
- Use TypeScript strict mode when possible
```

**Copy these instructions to:**
- VS Code: Settings → GitHub Copilot → Custom Instructions
- GitHub: https://github.com/settings/copilot
```

## Copilot-Specific Features

### Markdown Format

Copilot uses Markdown formatting for instructions. The skill automatically formats:

| Element | Format |
|---------|--------|
| Sections | `## Section Name` |
| Rules | Bullet points with `-` |
| Emphasis | `**bold**` or `*italic*` |
| Code blocks | ` ```language ` |
| Comments | `<!-- comment -->` |

### Platform Detection

The skill detects Copilot by looking for:

```bash
# Detection files
.github/copilot-instructions.md
.vscode/copilot-instructions.md
~/.config/github-copilot/instructions.md
```

### Language-Specific Rules

Copilot supports per-language instructions. Example:

```markdown
## Python
- Follow PEP 8 style guidelines
- Use type hints for all function signatures
- Prefer list comprehensions over map/filter

## TypeScript
- Use strict mode
- Prefer interfaces over type aliases for objects
- Use 'unknown' instead of 'any' for untyped values

## Go
- Follow effective Go guidelines
- Use goroutines for concurrent operations
- Handle errors explicitly, don't ignore them
```

## Integration Points

### VS Code + Copilot

```json
// .vscode/settings.json
{
  "github.copilot.enable": {
    "*": true,
    "yaml": false,
    "plaintext": false
  },
  "github.copilot.inlineSuggest.enable": true,
  "github.copilot.advanced": {
    "customInstructions": {
      "enabled": true,
      "path": ".github/copilot-instructions.md"
    }
  }
}
```

### GitHub Copilot CLI

```bash
# Install Copilot CLI
npm install -g @githubnext/github-copilot-cli

# Set custom instructions
export GITHUB_COPILOT_INSTRUCTIONS="$HOME/.config/github-copilot/instructions.md"

# Run analysis
/rulesmith

# Use with Copilot CLI
gh-copilot "implement a REST API" --instructions-file ~/.config/github-copilot/instructions.md
```

## Coding Standards

EOF

## Troubleshooting

### Instructions Not Applying

**Issue:** Copilot isn't using the custom instructions

**Solutions:**
1. Verify file location (`.github/copilot-instructions.md`)
2. Check Copilot is enabled in settings
3. Restart VS Code or browser
4. Verify instructions are valid Markdown

### Analysis Not Working

**Issue:** `/rulesmith` command not recognized

**Solutions:**
```bash
# Verify skill is installed
ls -la ~/.claude/skills/rulesmith/

# Check Copilot chat is active
# In VS Code: Ctrl+Shift+P → "GitHub Copilot: Start Chat"

# Try manual invocation
"Please analyze our conversation and suggest Copilot instruction improvements"
```

### Format Issues

**Issue:** Generated instructions don't display correctly

**Solutions:**
1. Ensure Markdown formatting is correct
2. Check for special characters that need escaping
3. Use proper code block syntax

## FAQ

**Q: Can I use this with both VS Code and GitHub web?**

A: Yes. Place instructions in `.github/copilot-instructions.md` for universal use.

**Q: How do I reset my instructions?**

A: Delete the instructions file or remove specific sections. Run `/rulesmith --rollback` to restore from backup.

**Q: Will this work with Copilot X (Claude-powered Copilot)?**

A: Yes. The skill generates platform-agnostic rules that work with any AI assistant.

**Q: Can I have different instructions for different repositories?**

A: Yes. Add `.github/copilot-instructions.md` to each repository. Repository-level instructions override global settings.
